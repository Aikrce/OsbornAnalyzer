# 奥斯本创新九问 · 智能分析工具

基于奥斯本检核表法的创新思维分析工具，集成75+真实案例数据库，支持本地分析和AI深度分析双模式。

## 🌟 功能特点

### 智能学习分析系统
- **本地案例数据库分析**：基于75+真实创新案例，无需API密钥即可使用
- **AI深度分析**：可选的DeepSeek AI增强分析，提供更个性化的创新建议
- **自学习案例数据库**：每次分析后自动保存新案例，系统越用越智能
- **智能降级机制**：AI分析失败时自动切换到本地分析，确保系统始终可用

### 专业分析框架
- **关键词深度分析**：从7个维度全面解构主题本质
- **九维度创新思考**：基于奥斯本检核表法的系统性创新分析
- **案例驱动建议**：每个维度提供3-5个具体可行的创新建议

### 用户体验优化
- **API密钥记忆**：首次输入后自动保存，无需重复填写
- **智能案例积累**：自动学习用户分析历史，优先推荐相关案例
- **可视化输出**：支持完整分析图、单张卡片、批量下载三种模式
- **响应式设计**：完美适配桌面和移动设备
- **深蓝科技风格**：现代化的毛玻璃效果界面设计

### 🧠 自学习机制
- **自动案例收集**：每次分析后自动保存创新建议到本地数据库
- **智能案例匹配**：优先使用用户历史案例，提高分析相关性
- **案例质量评分**：基于主题相关性、时间新鲜度等因素智能排序
- **数据库统计**：实时显示学习进度和各维度案例分布
- **隐私保护**：所有数据存储在浏览器本地，完全保护用户隐私

## 🎯 分析框架

### 第一步：关键词深度分析
系统从7个维度深度解构输入主题：
- **核心功能**：主要功能和价值主张
- **关键属性**：重要特征和属性标签
- **现有形态**：当前存在的主要形式
- **目标用户**：主要服务的用户群体
- **使用场景**：典型的应用场景和环境
- **价值链条**：在产业生态中的位置
- **约束限制**：面临的主要挑战和瓶颈

### 第二步：九维度创新分解
基于奥斯本检核表法进行系统性创新思考：

| 维度 | 核心思路 | 典型案例 |
|------|----------|----------|
| **能否他用** | 探索其他用途和应用场景 | 花生300种用途、X射线技术迁移 |
| **能否借用** | 借鉴其他领域的做法和原理 | 微爆破技术治疗肾结石、GPS动物追踪 |
| **能否改变** | 改变形态、流程、规则或属性 | 福特汽车颜色变化、平面镜变哈哈镜 |
| **能否扩大** | 扩大规模、功能、影响范围 | 药物牙膏、防弹玻璃创新 |
| **能否缩小** | 简化、专注核心功能、便携化 | 袖珍收音机、微型医疗器械 |
| **能否替代** | 替代材料、方法、技术或流程 | 纸质铅笔、植物基人造肉 |
| **能否调整** | 调整顺序、结构、流程或时间 | 飞机螺旋桨位置调整、生产线重组 |
| **能否颠倒** | 颠倒关系、反转思维、逆向操作 | 电动机发明、反向拍卖模式 |
| **能否合并** | 合并、组合、联动、集成 | 带橡皮铅笔、智能手机集成 |

## 🚀 快速开始

### 在线使用
1. 打开浏览器访问工具页面
2. 输入分析主题（如：智能咖啡机、在线教育平台等）
3. 点击"开始分析"按钮
4. 查看关键词分析结果
5. 切换到"九宫格编辑"页面查看创新建议
6. 可编辑内容并下载PNG图片

### 本地部署

#### 环境要求
- Node.js 14.0+
- 现代浏览器（支持ES6+）

#### 安装步骤
```bash
# 克隆项目
git clone <repository-url>
cd osborn-innovation-tool

# 安装依赖
npm install

# 启动服务器
npm start
```

#### 配置API密钥（可选）
创建 `.env` 文件：
```env
DEEPSEEK_API_KEY=your_deepseek_api_key_here
PORT=3000
```

### 简单部署
如果只需要前端功能，可以直接使用静态文件服务器：
```bash
# 使用Python
python3 -m http.server 8080

# 使用Node.js
npx serve . -p 8080
```

## 🔧 开发指南

### 项目结构
```
osborn-innovation-tool/
├── index.html          # 主页面
├── style.css           # 样式文件
├── app.js             # 主要逻辑
├── server.js          # 后端服务器（可选）
├── package.json       # 项目配置
├── .env              # 环境变量
└── README.md         # 项目文档
```

### 核心功能模块
- **案例数据库管理**：`osbornCaseDatabase` 对象
- **本地分析引擎**：`analyzeWithDatabase()` 函数
- **AI分析接口**：`analyzeWithAI()` 函数
- **智能降级机制**：自动切换分析模式
- **存储管理**：API密钥的安全存储

### 扩展开发
1. **添加新案例**：在 `osbornCaseDatabase` 中添加案例
2. **自定义分析逻辑**：修改 `generateSuggestionsFromCases()` 函数
3. **界面定制**：调整 CSS 变量和样式
4. **API集成**：支持其他AI服务提供商
